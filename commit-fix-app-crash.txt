fix(critical): Prevent app crash from zoneEnrollResponse unhandledRejection

CRASH CRITIQUE CORRIGÉ - App crashes completely

Error: "Impossible de joindre l'appareil. Est-il allumé ?"
Location: button_emergency_sos/device.js:59 (and 41 other drivers)
Crash type: unhandledRejection -> APP CRASH

ROOT CAUSE:

zoneEnrollResponse() returns a Promise
Without await, try/catch cannot catch the error
When device is sleeping/unreachable, Promise rejects
Unhandled rejection crashes the entire app

PROBLEMATIC CODE (42 drivers):
```javascript
try {
  endpoint.clusters.iasZone.zoneEnrollResponse({ // ← NO AWAIT!
    enrollResponseCode: 0,
    zoneId: 10
  });
  this.log('Sent');
} catch (err) {
  this.log('Failed'); // ← NEVER EXECUTED!
}
// Result: unhandledRejection -> APP CRASH
```

FIXED CODE (42 drivers):
```javascript
try {
  await endpoint.clusters.iasZone.zoneEnrollResponse({ // ← WITH AWAIT
    enrollResponseCode: 0,
    zoneId: 10
  });
  this.log('Sent');
} catch (err) {
  this.log('Failed'); // ← NOW PROPERLY CATCHES ERROR
}
// Result: Graceful error handling, NO CRASH
```

WHY THIS HAPPENS:

Battery devices sleep to save power
During app restart, many devices are sleeping
Proactive zoneEnrollResponse sent to all IAS devices
Sleeping devices cannot respond immediately
Without await, Promise rejection becomes unhandledRejection
Node.js crashes the app on unhandledRejection

TECHNICAL DETAILS:

Promises require await to be caught by try/catch:
- Promise chain: .then().catch() ✅ Works
- async/await: try { await promise } catch ✅ Works
- Without await: try { promise } catch ❌ DOESN'T WORK

This is a fundamental JavaScript async pattern requirement.
