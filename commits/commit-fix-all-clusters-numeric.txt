WHY THIS FIX:
SDK3 requires NUMERIC cluster IDs in registerCapability()
Using string cluster names causes: TypeError: expected_cluster_id_number

PATTERNS FIXED:
1. registerCapability('cap', 'clusterName', {...})
   → registerCapability('cap', numericID, {...})

2. this.registerCapability('cap', 'clusterName', {...})
   → this.registerCapability('cap', numericID, {...})

3. CLUSTER.propertyName
   → numericID
