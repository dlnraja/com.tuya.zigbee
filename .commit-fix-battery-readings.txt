fix: 🔋 Fix battery sensor readings (No data since v4.5.4)

🐛 PROBLÈME CRITIQUE RÉSOLU:

SYMPTÔMES RAPPORTÉS (Peter + diagnostics):
❌ "No Data readings anymore from Multisensor" (v4.5.4/4.5.5/4.5.6)
❌ Motion sensor multi: temperature, humidity, luminance non mis à jour
❌ Battery monitoring setup failed: "expected_cluster_id_number"

ROOT CAUSE:
BaseHybridDevice.setupBatteryMonitoring() utilisait configureAttributeReporting()
avec une syntaxe incorrecte qui causait une erreur silencieuse.
L'erreur bloquait l'initialisation complète du device, empêchant
les lectures de capteurs (temperature, humidity, luminance, motion).

SOLUTION:
Supprimé configureAttributeReporting() du battery monitoring.
La registration des capabilities suffit - Homey gère automatiquement
le reporting via registerCapability().

CODE MODIFIÉ:
- lib/BaseHybridDevice.js: Simplifié setupBatteryMonitoring()
- Removed: configureAttributeReporting([...])
- Kept: registerCapability() pour measure_battery
