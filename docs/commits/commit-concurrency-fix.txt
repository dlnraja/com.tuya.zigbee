fix: Add concurrency control to prevent workflow conflicts (SOLUTION DÉFINITIVE)

PROBLÈME RÉSOLU:
Workflows GitHub Actions échouaient avec "push rejected" car plusieurs workflows
s'exécutaient simultanément et tentaient de pusher en même temps.

SOLUTION IMPLÉMENTÉE - Concurrency Control:
Ajout de concurrency groups à tous les workflows critiques pour garantir qu'un
seul run est actif à la fois par workflow.

Changements:
1. .github/workflows/update-docs.yml
   - Ajout concurrency group: workflow-ref
   - cancel-in-progress: true
   - Empêche runs concurrents

2. .github/workflows/homey-official-publish.yml  
   - Ajout concurrency group: workflow-ref
   - cancel-in-progress: true
   - Empêche runs concurrents

Comportement:
AVANT: 3 commits rapides → 3 workflows concurrents → 2 push rejected
APRÈS: 3 commits rapides → workflow 3 seul actif (1&2 cancelled) → 1 push success

Avantages:
✅ Élimine race conditions
✅ Optimise GitHub Actions minutes (annule runs obsolètes)
✅ Toujours run le plus récent
✅ Solution standard GitHub (maintainable)
✅ Pas de code custom requis

Expected Result:
- Workflows ne se bloquent plus
- Dernier commit = priorité
- Pas de push rejected
- Runs annulés auto si obsolètes
- CI/CD pipeline fluide
