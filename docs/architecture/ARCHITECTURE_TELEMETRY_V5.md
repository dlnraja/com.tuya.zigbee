## 1. Core Principles

### 1.1 Unified Tunneling
All capability updates, whether originating from Tuya DP reports, ZCL attribute reports, or virtual button events, MUST pass through a single, unavoidable tunnel: `TuyaZigbeeDevice.setCapabilityValue()`.

### 1.2 Multi-Layer Hardening (L14)
The telemetry pipeline implements three layers of protection before a value reaches the Homey SDK:
1.  **SanityFilter (EMA/ROC)**: Filters out sensor noise and transient spikes.
2.  **Anti-Flood Throttling**: Prevents high-frequency reports from saturating the Homey Event Loop.
3.  **Significant Change Detection**: Only propagates updates that exceed a defined threshold (or after a maximum quiet time).

## 2. Technical Implementation

### 2.1 The Hardened Base Class

```javascript
// Simplified Logic Pattern
async setCapabilityValue(capability, value) {
  // 1. Sanity Filtering (EMA/ROC)
  if (!this.passesSanityFilter(capability, value)) return false;

  // 2. Significant Change & Throttling

    return false;
  }

  // 3. Propagation to Homey SDK

}
```

## 3. Specialized Overrides
