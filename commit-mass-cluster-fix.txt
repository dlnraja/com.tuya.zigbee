MASS CONVERSION - 140 DRIVERS UPDATED
======================================

CRITICAL SDK3 REQUIREMENT
==========================
Homey SDK3 requires using CLUSTER constants from zigbee-clusters library.
Numeric cluster IDs cause "TypeError: expected_cluster_id_number" runtime errors.

ROOT CAUSE
==========
Drivers were using legacy numeric cluster IDs:
❌ this.registerCapability('measure_battery', 1, {...})
❌ cluster: 1026,
❌ this.registerCapability('onoff', 6, {...})

SDK3 REQUIREMENT
================
Must use CLUSTER constants from zigbee-clusters:
✅ this.registerCapability('measure_battery', CLUSTER.POWER_CONFIGURATION, {...})
✅ cluster: CLUSTER.TEMPERATURE_MEASUREMENT,
✅ this.registerCapability('onoff', CLUSTER.ON_OFF, {...})

AUTOMATED FIX
=============
Created script: scripts/fixes/FIX_ALL_CLUSTER_IDS.js
- Scans all drivers/*/device.js files
- Converts numeric IDs to CLUSTER constants
- 140 files automatically corrected
